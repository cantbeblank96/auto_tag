## 测试目的

验证标注任务页的完整提交流程：

1. 通过「上传 JSON」将任务配置加载到「新建」表单；
2. 表单字段与 JSON 内容一致；
3. 「确认」将任务加入队列；
4. 「提交任务」触发后端流水线并成功完成。

## 操作步骤

### 步骤 1：打开标注任务页

1. 浏览器访问 http://localhost:5020
2. 确认左侧导航高亮 **「标注任务」**
3. 页面包含三个区域：**加载 & 保存**、**新建**、**运行**

### 步骤 2：上传 JSON 加载到表单

1. 在 **「加载 & 保存」** 区域，点击 **「上传 JSON 加载到表单」**
2. 选择文件：`auto_tag/test/auto_tag_job.json`（使用本机绝对路径）
3. 等待页面顶部出现提示：**「已加载到表单」**

### 步骤 3：核对「新建」表单

对照下表检查 **「新建」** 区域是否已正确填充：

| 表单项 | 预期值 |
|--------|--------|
| 输入目录 | `.../auto_tag/test/test_data`（与 JSON 中 `input_dirs` 一致，每行一个路径） |
| rotate_angle | **逆时针 90° (ROTATE_90_COUNTERCLOCKWISE)** |
| 混合目录 | **已勾选**（`mixed_yuv: true`） |
| 整批均为 YUV | **未勾选**（`b_yuv_image: false`） |
| YUV 宽度 | **178** |
| YUV 高度 | **218** |
| YUV 类型 | **nv21** |

**通过标准**：上述字段与 JSON 完全一致；出现「已加载到表单」提示。

### 步骤 4：确认加入队列

1. 点击 **「新建」** 区域底部的 **「确认」**
2. 页面提示：**「已加入任务队列，请在「提交任务」中执行」**
3. 在 **「运行」** 区域队列表中出现一行新任务：
   - **状态**：排队中
   - **摘要**：显示 `test_data` 目录路径

**通过标准**：队列表有且仅有一条（或新增一条）状态为「排队中」的任务；**「提交任务」** 按钮变为可点击（非灰色禁用）。

### 步骤 5：提交并执行任务

1. 在 **「运行」** 区域，确认 **「跳过库中已有路径」** 可按测试需要勾选或取消（默认勾选；首次跑库时一般无影响）
2. 点击 **「提交任务」**
3. 观察队列表与进度：
   - 状态变为 **运行中**，随后变为 **已完成**（或 **失败**）
   - **已收集** 应为 **7**
   - 进度条（若有）走至 100%

**通过标准**：任务最终状态为 **已完成**，无报错弹窗。

### 步骤 6（可选）：API 交叉验证

在终端执行：

```bash
curl -s http://127.0.0.1:8000/api/jobs | python3 -m json.tool
```

找到最新一条 `job`，核对：

| 字段 | 预期（首次全量跑 test_data、且库中无同路径记录时） |
|------|------------------------------------------------------|
| `status` | `done` |
| `total` | `7` |
| `processed` | `7` |
| `failed_so_far` | `0` |
| `vlm_calls` | 约 `1`（首张或新簇触发 VLM） |
| `stage1_skips` | 约 `6`（其余近重复帧 Stage 1 跳过） |
| `skip_in_db` | `0`（首次跑库；若重复提交且勾选跳过则可能 > 0） |

也可查看单任务详情：

```bash
# 将 {job_id} 替换为实际 ID
curl -s http://127.0.0.1:8000/api/jobs/{job_id} | python3 -m json.tool
```

| 列 | 典型预期 |

| 打标数 | 1 (14%) 左右 |

### 业务含义（供测试人员理解）

测试集内多张图视觉高度相似，流水线行为符合设计：

- **Stage 1**（`dist ≤ tau_dup`）：近重复图不入库，计入 `stage1_skips`，并写入侧车（若开启）
- **Stage 3**（新簇）：至少 1 次 VLM 打标，计入 `vlm_calls`

具体数值可能因历史索引、阈值配置略有差异；**失败数为 0 且状态为已完成** 为硬性通过条件。

## 失败与排查

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 「提交任务」一直禁用 | 未点击「确认」或队列无「排队中」任务 | 先完成步骤 4 |
| 状态「失败」 | VLM API 不可用、CLIP/CUDA 异常 | 查看 `error` 字段或 `/api/jobs/{id}/logs` |
| 已处理 < 7 | 输入目录路径错误 | 检查 JSON 中 `input_dirs` 是否为本机绝对路径 |
| 打标数为 0 且跳过数很少 | 索引中已有数据且勾选「跳过库中已有路径」 | 取消勾选后重跑，或清空 `work_dir/embedding_index` 后重测 |
| 前端无法访问 | 服务未启动或端口不对 | 确认 5020 / 8000，执行 README 健康检查 |
| 上传 JSON 无反应 | 文件格式错误或路径无读权限 | 确认 `.json` 合法、浏览器允许选文件 |
